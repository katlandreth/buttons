Limitations

There are a few things you should know about.

  1. The button corners should not be rounded. A browser quirk will render white lines inside the border on hover when corners are rounded.
  2. You can use unicode characters and some icon fonts as the button content . Icon-fonts like Ionicon that make use of unicode's private use area may not work correctly.
  3. The hover content must be exactly the same as the original button content.
  4. Other CSS files in your project could potentially interfier with your button styles. If that's happening, check the troubleshooting section at the end of this help document.
  5. These buttons are intended to use with the Button element. Their use on other elements is not supported.

Setting Up

Before you can start using your buttons you'll need to set up the CSS file and install the Google Font "Lato".

TL;DR - Setup

Already familiar with HTML and CSS? Here's a summary of the setup steps. If you need more help, more detailed instructions follow.

  1. Place the buttons.css file in the appropriate directory of your site.
  2. Link to the buttons.css file in the Head of your site.
  3. Include the Google Font "Lato" with weights 300 and 400.
  4. Start using your buttons!

More Detailed Instructions

First thing's first. Download and unzip your purchase from Envato. Once unziped, you'll have a main Buttons folder with a bunch of other files and folders inside it. Most of that is documentation. The only file you need to add to your website is the one titled "buttons.css".

Upload this file to the appropriate section of your website, or if you're working on a local version of the site (always a good idea) just copy the buttons.css file into the appropriate local folder.

Not sure where to put your buttons.css file? You can usually put your stylesheet anywhere you want, as long as you can write a path to the file (see below). Some frameworks and CMS's, however, have conventions about where you should put css files. For example, in Wordpress your theme might have a folder called CSS for your css files. That might look something like this:

If you're not sure where to put your css files, do an internet search for "where to put style sheets in _____" filling in the blank with the name of your framework or CMS.

Once buttons.css is in the right place, all you have to do is link to the style sheet in the Head of your HTML document (between the "head" tags at the top of the HTML document) like so:

<head>
<link rel="stylesheet" type="text/css" href="css/buttons.css">
</head>

The example above uses a relative path to reference the css file. You can write out the whole path, but it's more future-proof to use a relative path.

This relative link tells you where the css file is relative to the location of the HTML file. The relative path above means that buttons.css is in a folder called "css" that's in the same level as the HTML document.

For more help with setting up the correct relative path to your buttons.css file, please see this great article on CSS Tricks - Quick Reminder About File Paths

Using the Buttons

The buttons are intended to be used on the button element with the instructions in this section. You can technically use these button styles on other elements, but the results may not be as expected, and using these classes on anything other than the button element is not supported.

TL;DR - Using Classes

To use a flat button, you always start by adding the flat-button class. Color classes and size classes can be added with or without hover effect classes. Add a directional hover class if you want to use hover effects. You must add the data-text attribute to the html button if you're using hover effects. Your final button code might look something like this:

<button class="flat-button fb-top fb-warning fb-alt-hover" data-text="Buy Now">Buy Now</button>

Class Cheat-Sheet

Here's a quick cheat-sheet of the various button classes for those of you who don't need a lot of instruction. Remember, it's reccommended that you use these classes on a button element as they won't work as expected on other input elements.

  • flat-button
    sets the base styles for the button and is needed for all the other classes. There are no hover effects for flat-button.
  • fb-top
    , 
    fb-bottom
    , 
    fb-left
    , and 
    fb-right
    initialize a hover effect with a coorisponding hover origin, e.g. fb-top causes the hover state to slide in from the top.
  • fb-warning
    , 
    fb-attention
    , 
    fb-callout
    , 
    fb-info
    , 
    fb-go
    , and 
    fb-note
    change the colors of the buttons. The default color is light teal and needs no additional class to initialize.
  • fb-small
    , and 
    fb-big
    change the size of the buttons. The default size is "medium", or more specifically 1.5em. 
  • fb-alt-hover
    activates the alternative hover effect which is a solid color background with white text.
  • data-text
    attribute must be added to the button html with a value exactly equal to the content of the button if you're using hover effects. See the example below.

Further Instructions - Classes

Got all that? Here are some more detailed instructions just in case.

The Default Button

The simplest way to start using flat buttons is to add the flat-button class to an HTML button like this:

<button class="flat-button">Buy Now</button>

Which would render the following button:

The default flat button (above) is a medium sized button with transparant background, and light teal text and border. It will have no hover effects.

You'll need to use the flat-button class every time you use a flat button, even when you add other fb classes. To include hover effects, you must use additional hover and modifier classes.

Hover Effects

Use the base class flat-button and an aditional hover class of either fb-top, fb-bottom, fb-left, or fb-right to control the hover effect of the button. You must also include the data attribute data-text with the value exactly equal to the content of the button in order for the hover effects to work properly. For example:

<button class="flat-button fb-top" data-text="Buy Now">Buy Now</button>

Modifyers

There are a few ways to modify your flat buttons. You can change their size, color, and use an alternative hover effect by simply adding extra classes to the button.

Sizes

There are three sizes available for the button. The default medium size is built into the base class. No extra class is needed to use the medium size buttons.There is also a fb-small class, and a fb-big class, which will make the button smaller, or larger respectively. 

<button class="flat-button fb-top fb-big" data-text="Buy Now">Buy Now</button>

Which would render the following button:

The classes fb-small and fb-big must be used in conjunction with the flat-button class to work properly.

Colors

There are seven colors available for the buttons. There's the default light teal, and six other colors that can be used by adding classes to the buttons. The class names are based on the kind of thing you might use that color for. This way, if you choose to change the colors inn the future, the class names will still make sense. The color classes are:

  • fb-warning
  • fb-attention
  • fb-callout
  • fb-go
  • fb-info
  • fb-note

Add a color class to the button markup to use a color other than the default, like this:

<button class="flat-button fb-top fb-alt-hover fb-warning" data-text="Buy Now">Buy Now</button>

Which would render the following button:

Remember, you must use the flat-button class with a color class to make the color classes work properly.

The Disabled State

When buttons are disabled, they will automaticaly lose all hover effects, and become a soft gray color.

To disable a button, simply add "disabled" to the button like this:

<button class="flat-button fb-top fb-alt-hover fb-warning" data-text="Buy Now" disabled >Buy Now</button>

Altering the Buttons with SASS & CSS

Modifying the CSS using SASS

These buttons were initially created using SASS (Syntactically Awesome Style Sheets) which is a css preprocessor that makes CSS simpler to write and maintane. While you should only need to include the actal CSS file (buttons.css) the original SASS file (buttons.css.scss) is also included with this package to make updating and editing your buttons easier. The size and colors of the buttons are easily changed in the SASS file. If you're new to SASS, I recommend downloading and installing SCOUT which will help you preprocess your SASS file and generate an updated CSS file that you can use on your website once you've completed any updates or revisions.
